# Plan: Besserer Live-Watcher im Harness Dashboard (Port 8420)

## Goal
Samuel kann im Dashboard auf http://127.0.0.1:8420/ live verfolgen was Aider macht — in Echtzeit, ohne auf meine Reports angewiesen zu sein.

## Current Context
- Dashboard auf Port 8420, Flask/FastAPI + Vanilla-JS
- **Live-Tab**: Pollt `/api/status` alle 5s → zeigt nur Tabellenergebnisse (Status, Tests, Duration)
- **Monitor-Overlay**: SSE-Stream von `/api/runs/{runId}/stream` → kann Aider-Output + Patch + Events live anzeigen
- **Problem**: Monitor öffnet sich NUR wenn Run vom Dashboard aus gestartet wird (app.js:190). Bei CLI-Runs muss Samuel manuell auf "Monitor" im Banner klicken. Ausserdem: Live-Tab zeigt KEINE Live-Logs.

## Proposed Approach
**Zwei Optionen:**

### Option A: Monitor inline in Live-Tab integrieren (einfacher)
Live-Tab zeigt für den aktuell laufenden Run einen expandierten Bereich unter der Tabelle mit:
- Echtzeit-Aider-Log-Output (stdout gestreamt via SSE)
- Patch-Vorschau
- Phase + Timer

### Option B: Auto-open Monitor Overlay bei laufendem Run
Wenn Live-Tab einen laufenden Rundetektiert → automatisch Monitor öffnen (kein Klick nötig)

## Step-by-Step Plan

### Step 1: Verstehen was schon existiert
- [x] `server.py` — `/api/runs/{run_id}/stream` SSE endpoint existiert (line 431)
- [x] `app.js` — `openMonitor()` mit SSE-Client existiert (line 200)

### Step 2: Live-Tab erweitern — laufenden Run inline streamen
**`app.js`**: Neue Funktion `renderLiveMonitor()` in `loadLive()`:
- Prüfe `running.running`
- Wenn ja: öffne SSE zu `/api/runs/{running.run_id}/stream`
- Zeige Output in einem `<pre>` unter der Tabelle
- Auto-Scroll, Token-Counter, Phase-Badge

**`index.html`**: Neues `<div id="live-monitor-panel">` unter `<table id="live-table">`

**`style.css`**: Styling für live-monitor-panel

### Step 3: Token-Counter aus Aider-Output extrahieren
Aider gibt Token-Zähler in stderr aus (litellm Format):
```
tokens_in: 1234, tokens_out: 456, cost: $0.00123
```
- [ ] Regex in `app.js` beim Stream: Token-Zahlen aus Log-Zeilen extrahieren
- [ ] Anzeige: "In: 1234 | Out: 456 | Cost: $0.001 | Running: 2m 30s"

### Step 4: Phase-spezifisches Verhalten
- [ ] `aider_running` → Zeige "Aider denkt..." pulsing indicator
- [ ] `eval_running` → Zeige "Tests laufen..."
- [ ] `done` → Auto-Refresh der Tabelle, grüner Flash

### Step 5: CLI-Integration verbessern
Runner-Process informiert Dashboard nicht automatisch über neue Runs.
- [ ] `runner/run_once.py`: POST an Dashboard wenn Run startet (optional)
- [ ] Oder: Dashboard pollt aktiv neue Runs aus results/ Verzeichnis

## Files likely to change

## Risks & Tradeoffs
- SSE-Verbindung braucht CORS-Handling wenn Dashboard + Runner auf verschiedenen Ports
- Token-Counter nur so genau wie Aider sie in Logs schreibt
- Auto-Monitor könnte irritieren wenn Samuel gleichzeitig CLI-nutzt
