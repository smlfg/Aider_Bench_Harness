Aider Bench Harness — Logic Flow

Task + .md → Agent → Patch → Judge → Verdict

Step 1: Task

  • • SWE-bench instance with real GitHub issue
  • • problem_statement: describes the bug
  • • FAIL_TO_PASS: tests that must pass after fix
  • • PASS_TO_PASS: tests that must still pass
  • • test_patch: expected test file changes

Step 2: CONVENTIONS.md

  • • Guides agent behavior with coding rules
  • • Baseline: default coding style
  • • Candidate: experimental rules to test
  • • Located in harness/CONVENTIONS.*.md
  • • Compared in experiments

Step 3: Judge

  • • Judge LLM evaluates the patch
  • • Two-stage: scope + minimality
  • • Separates infra failures from real failures
  • • SWE-bench eval runs actual tests in Docker
  • • Verdict: pass / fail / mixed
