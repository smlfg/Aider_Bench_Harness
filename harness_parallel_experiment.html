Parallel Experiment — Up to 100 Runs

Tasks × Conditions × Runs → Parallel Pipeline Instances → Aggregated Results

Experiment Matrix

  • • Tasks × Conditions × Runs = Total runs
  • • Example: 5 tasks × 3 conditions × 5 runs = 75 runs
  • • Up to 100 runs in parallel (parallelism limit)
  • • Each run: independent run_once.py process
  • • repo_cache shared across runs (read-only)

Aggregation

  • • summarize.py: per task × condition stats
  • • experiment_summarize.py: baseline vs candidate
  • • SQLite DB: all run_meta persisted
  • • Web UI: monitor (live) + debrief (compare)

What Gets Compared

  • • pass_rate: how often did agent succeed?
  • • avg_judge_score: how good was the patch?
  • • delta_pass_rate: candidate minus baseline
  • • delta_avg_judge_score: candidate minus baseline
  • • Statistical significance of the difference
