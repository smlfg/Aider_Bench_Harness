Das Experiment

Ziel

Kann man den Effekt kleiner Änderungen an einer Markdown-Instruktionsdatei (CONVENTIONS.md) bei einem einfachen Coding-Harness belastbar messen?

Das Experiment ist absichtlich klein und falsifizierbar. Es soll klären, ob die Methode lebt oder stirbt.

Setup

  • Harness: Aider
  • Modell: MiniMax-M2.7-highspeed
  • Tasks: 3 Python-Bugfix-Tasks aus SWE-bench Lite
  • Runs: 5 pro Task pro Condition (Baseline + Candidate)
  • Optimierungsobjekt: Genau eine Datei — CONVENTIONS.md

Metriken

Statistik

  • Wilson-CI für task_success (Bernoulli-Intervall)
  • Bootstrap-CI für stetige Metriken (pass_rate, diff_size, duration)
  • Fisher's Exact Test für task_success (Baseline vs. Candidate)
  • Mann-Whitney U für Ordinal-/Stetigvergleiche
  • Cliff's Delta als Effect Size
  • Pareto-Frontier: Iterationen dominieren, wenn success_rate gleich oder höher UND diff_size gleich oder kleiner

--skip-eval

Evaluierung wird übersprungen. Kein Docker nötig.

  • tests_total=0, task_success=False
  • Patch wird trotzdem gespeichert (git_diff.patch)
  • Eval kann nachträglich nachgeholt werden
  • Nützlich für schnelle Iteration oder wenn Docker nicht verfügbar

Judge

Separater Schritt (harness-judge), nicht Teil von run_once.

  • Bewertet Code-Qualität: Scope Adherence, Minimality, Diff Clarity (je 1-5)
  • Braucht JUDGE_COMMAND in .env
  • Ohne Command: --allow-stub → Neutral-Score 3.0

Hinweise

  • Doppelter Eval-Bug: Bei normalem Lauf wird eval zweimal getriggert (Performance-Problem, kein Korrektheits-Problem — zweiter Lauf findet gecachten Report)
  • tokens_in = 0 → Infrastructure Error (Modell-Auth, Routing, etc.)
  • infrastructure_error = true → Run fließt nicht in analyze.py-Statistiken ein
  • Error-Detail: Infra-Fehler zeigen im Debrief jetzt den konkreten Fehler-Snippet (z.B. authorized_error (1004))
